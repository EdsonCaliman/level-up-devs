Quando execuções paralelas são colocadas em prática, seja na mesma máquina ou em máquinas diferentes, vários problemas podem ocorrer devido à acessos indevidos à recursos compartilhados ou execuções mal sincronizadas. 1. O gerenciamento da memória de um software deve ser feito de forma manual em linguagens de programação mais antigas. O conhecimento sobre esse assunto é importante para que o desenvolvedor possa otimizar o consumo de recursos computacionais.
#### Aprender sobre profilers
1. Profilers são ferramentas para permitir a "telemetria" de um software em execução. Em linhas gerais o profiler é anexado à um software em execução que inspeciona tempos de execução de métodos, consumo de memória ao longo do tempo, vazamentos de memória, dentre outros.

## Trilha: Testabilidade
### Objetivo: Conhecer os tipos de testes automatizados existentes e entender os conceitos de mocks e stubs (Nível 1)
#### Mocks, spies e stubs
1. Os testes de unidade são chamados assim por testar "unidades" de código. Visando a didática, uma unidade de código pode ser resumido à um método ou uma função. Para manter o teste isolado como se propõe, todos os recursos utilizados pela tal unidade de código, não poderão ser os recursos de produção (reais). Mocks, spies e stubs são utilizados como dublês de recursos reais, permitindo validar interações ocorridas nesses recursos
#### Conhecer a biblioteca de testes utilizada no projeto em que atua
1. A execução de testes automatizados se dá por meio do uso de bibliotecas para este fim. Cada biblioteca oferece um conjunto específico de recrusos e maneiras específicas para sua utilização. O conhecimento aprofundado na biblioteca que utiliza permitirá explorar recursos que facilitarão a escrita/execução de testes automatizados

#### Cobertura de código

#### TDD
1. Sigla para Test Driven Development, o TDD é uma metologia de escrita de sistemas em que o foco é a testabilidade, chegando inclusive na máxima em que os testes são escritos antes mesmo da existência dos códigos que testarão

1. Atomicidade é um conceito de bancos de dados que descreve o agrupamento de múltiplas operações que devem ocorrer de forma indivisível: ou todas funcionam ou todas falham. O controle transacional é o meio pelo qual transações atômicas são gerenciadas

1. Em bancos de dados relacionais, índices são estruturas que otimizam a busca dos dados e as constraints são regras que os programadores podem definir para garantir a consistência de dados

1. ACID é um acrônimo para Atomicidade, Consistência, Isolamento e Durabilidade que são propriedades de transações de bancos de dados
